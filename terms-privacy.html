Password & Security System

This application is protected by a password entry screen. The purpose of this security screen is to prevent unauthorized persons from accessing your payment information and UPI details stored on your device.

How the security system works:

  • You are given 3 attempts per cycle to enter the correct password.
  • After 3 wrong attempts, the app enters a lockout period.
  • Lockout durations increase progressively: 1st lock = 15 seconds, 2nd lock = 30 seconds, 3rd lock and beyond = 60 seconds.
  • After each lockout expires, you receive 3 new attempts.
  • The password and lockout state are tracked using browser localStorage.
🔑 Important: The password for this application is stored as plain text in the JavaScript source code. This is a client-side application and therefore security depends on keeping the password confidential and not sharing the source code publicly with the password visible.

You are responsible for keeping your password confidential. Do not share the application password with untrusted individuals. If your device is lost or shared, anyone with access to the browser may be able to inspect the source code.

How & Where Data Is Stored

All data in Imran X Dynamic is stored exclusively using browser localStorage — a standard web technology that stores data only on your device, in your browser.

Key facts about localStorage storage:

  • Data is stored only on the specific device and browser you are using.
  • Data is never sent to any server operated by us.
  • Data persists across sessions — meaning if you close and reopen the browser, your name and UPI ID are still there.
  • Data is deleted if you clear your browser's data, uninstall the browser, or use the "Clear Site Data" function.
  • Data is not synced across devices. If you use this app on two different phones, each has its own separate data.
  • Other websites and apps cannot read your localStorage data. It is sandboxed per domain.
ℹ️ To delete your data at any time: Go to your browser Settings → Privacy & Security → Clear browsing data → Site Data → Find this website → Clear. This will erase all saved username, UPI ID, and lock state data.

The localStorage keys used by this application are:

  • ixd_username — stores your display name
  • ixd_upiid — stores your UPI ID
  • ixd_locked_until — stores the timestamp of the lockout expiry
  • ixd_lock_count — stores how many times lockout has occurred

UPI Links, Payments & Financial Data

This application generates UPI deep links — a standard format recognized by all UPI-enabled apps in India (Google Pay, PhonePe, Paytm, BHIM, etc.). These links are generated entirely within your browser using JavaScript.

How UPI links are generated:

  • We take your UPI ID, name, and amount that you provide.
  • We assemble them into the standard UPI URL format: upi://pay?pa={UPI_ID}&pn={NAME}&am={AMOUNT}&cu=INR
  • This assembly happens entirely in your browser. No server is involved.
  • The generated link is then either: displayed as text, encoded into a QR code (via external API — see Section 9), or shared via your device's share function.
⚠️ Important Financial Disclaimer: Imran X Dynamic does NOT process, initiate, facilitate, or authorize any financial transaction. We are purely a link and QR generation tool. Any payment made using a generated UPI link is processed entirely by the receiving bank and UPI infrastructure (NPCI). We have zero involvement in, or liability for, any transaction.

We strongly advise:

  • Always double-check your UPI ID before sharing a generated link or QR.
  • Only share your payment QR with people you intend to receive payment from.
  • Never enter someone else's UPI ID without their explicit consent.
  • Do not use this app to facilitate any unauthorized or fraudulent transaction.

QR Code Generation & Third-Party APIs

When you click "Generate QR Code", this application sends a request to a third-party API service:

🌐 QR API Used: https://api.qrserver.com/v1/create-qr-code/

Operated by: goQR.me — a public, free QR code generation service.

What data is sent to this API:

  • The encoded UPI link (which contains your UPI ID, name, and amount) is sent as a URL parameter to the QR API server.
  • The server generates a QR image from this data and returns it to your browser.
  • The requested image size (300×300 pixels) and error correction level (M) are also sent.
⚠️ Please be aware: When you generate a QR code, your UPI ID, name, and the specified amount are transmitted as URL query parameters to the goQR.me server. This is necessary for them to create the QR image. Their server may log request data per their own privacy policy. Visit goqr.me to review their privacy practices.

We have no affiliation with or control over goQR.me. We are not responsible for how they handle request data. If you have concerns, you may choose not to use the QR generation feature and instead use only the UPI Link sharing feature, which involves no third-party servers.
